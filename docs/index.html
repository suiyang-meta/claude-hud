The problem

You're mid-conversation with Claude and you wonder: am I about to get rate-limited?

You open a new tab. Go to settings. Scroll to usage. Stare at the numbers. Close the tab. And by the time you're back in Claude, you've lost your train of thought.

Claude HUD makes that moment disappear.

What it does

A small floating window sits in the corner of your screen showing four color-coded bars: current session, weekly all models, weekly Sonnet, and daily routine runs. It updates every 30 seconds. Green under 50%, yellow 50–79%, red at 80%+.

You glance at it. You go back to work.

Always on top
Floats over every window on every workspace. 70% opacity when idle, 100% on hover.
Launches at login
Set it once. It just shows up whenever your Mac is on.
Drag, resize, hide
Put it wherever you want. Ghost mode fades it to 5% opacity when you need focus.
Your data stays local
No servers, no analytics, no accounts. Everything happens on your machine over localhost.

How it works

Claude HUD is two pieces: a Mac app (the floating window) and a Chrome extension (the data pipe). The extension quietly keeps a tab open at claude.ai/settings/usage, refreshes it every 30 seconds, reads the four usage numbers off the page, and pushes them to the Mac app over localhost port 27843.

Nothing ever touches the internet beyond Chrome → claude.ai, which was already happening anyway. Your conversations, prompts, and account data are never read.

FAQ

Is this affiliated with Anthropic?
No. I'm an independent developer who got tired of clicking into settings every 20 minutes. This is a third-party tool.
Can it read my conversations?
No. The extension only reads the numeric percentages on claude.ai/settings/usage. It has no access to your chats, prompts, or account.
Why a background Chrome tab?
Because Anthropic doesn't publish a usage API. The only way to get the numbers is to scrape the usage page. Manifest V3 doesn't allow truly hidden tabs — but you can pin it so it stays out of your way.
Does this use the Claude API?
No API calls. No API key required. It just reads the web page you'd see yourself.
Is it safe to run?
The code is on GitHub — read it yourself. The Mac app is unsigned (no $99/yr Apple Developer account on a side project), so macOS will ask you to explicitly allow it on first launch. That's normal.
Will it break if Anthropic changes the page?
Possibly. The extension uses regex to find the usage numbers. If Anthropic rewords the page, the numbers won't load until I push an extension update. Email me if that happens.
Why pay-what-you-want?
I built it for me. Sharing it costs nothing extra. If it saves you the context-switch and you want to throw a few bucks in, great. If not, also fine.
